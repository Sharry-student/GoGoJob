# 传媒行业招聘数据分析平台 PRD 文档

## 一、项目概述

### 1.1 项目背景

媒体专业求职者对中国大陆行业就业市场缺乏直观、数据化的了解，求职过程中存在信息不对称、决策效率低等问题。本平台通过预先爬取好智联招聘岗位数据，提供多维度数据分析、薪资预测与智能岗位匹配功能，帮助媒体专业求职者高效洞察市场现状、优化求职决策。

### 1.2 项目目标

- 为媒体专业求职者提供岗位薪资、技能需求、学历/经验要求等多维度数据洞察；

- 实现基于机器学习的薪资预测与多策略岗位匹配；

- 通过可视化界面降低信息获取门槛，提升求职效率。

### 1.3 目标用户
本项目主要面向媒体专业求职者（含应届生与从业者），旨在通过数据分析降低行业信息差。

平台核心数据库涵盖 14 个细分岗位，划分为两大专业类别：
- **内容生产 & 运营**：
编辑、内容运营、广告、公关、调研分析、产品经理、政府事务。

- **商业增长 & 支持**：
市场营销、推广投放、电商运营、线下运营、业务运营、销售招聘、人力资源。
## 二、核心功能

### 2.1 招聘数据分析

- **功能描述**：支持按职位类别、地区筛选，多维度展示岗位数据分布。

- **分析维度**：

    - 薪资分布：分位数统计、薪资区间占比；

    - 学历/经验要求：学历（本科/硕士等）、经验（1-3年/3-5年等）占比；

    - 技能需求：高频技能词云、技能共现关系。

- **输出形式**：ECharts 可视化图表（柱状图、饼图、词云）。
- ### 2.2 职位详情分析

- **功能描述**：针对特定职位（如“新媒体运营”）进行深入趋势分析。

- **分析内容**：

    - 薪资趋势：近6个月薪资变化曲线；

    - 技能需求：Top10 高频技能及占比；

    - 公司分布：热门行业、公司规模占比。

- **输出形式**：趋势图、统计表格。
### 2.3 地区薪资地图

- **功能描述**：可视化展示全国各地区薪资水平与职位分布。

- **展示内容**：

    - 地图热力图：各省份平均薪资；

    - 气泡图：各城市职位数量与薪资水平。

- **交互逻辑**：支持点击地区查看该地区岗位详情列表。
### 2.4 岗位交叉对比
- **功能描述**：支持用户选择两个不同的“岗位类别 + 工作城市”组合，进行多维度薪资数据的横向可视化对比。

- **对比维度**： 
  - 核心薪资：直观展示两组组合的平均月薪数值对比。 
  - 薪资结构：同步展示两组组合各自的最低、平均、最高薪资分布趋势。

- **输出形式**：
  - 数据卡片：高亮显示具体的薪资数值。 
  - 可视化图表：使用 ECharts 柱状图联动展示两者的薪资水平差异。

### 2.5 薪资预测

- **功能描述**：基于用户输入的个人条件，预测可能的薪资区间。

- **输入项**：职位方向、地区、学历、工作经验、技能组合、公司规模。

- **输出项**：预测薪资区间（如 15k-20k）、模型置信度。

- **实现逻辑**：加载预训练机器学习模型，实时计算并返回结果。

### 2.6 岗位匹配

- **功能描述**：根据用户技能、期望地区等条件，智能匹配岗位并支持多策略排序。

- **输入项**：个人技能、期望地区、期望职位、期望薪资。

- **匹配维度**：

    - 技能匹配度：用户技能与岗位要求的重合度（Jaccard 系数）；

    - 地区匹配度：期望地区与岗位地点的一致性；

    - 岗位方向匹配度：期望职位与实际岗位的相关性；

    - 薪资匹配度：岗位薪资是否落入用户期望区间。

- **优先策略**：综合匹配、技能优先、地区优先、薪资优先（动态调整各维度权重）。

- **输出项**：按匹配度排序的岗位列表，含岗位详情、各维度匹配得分。

## 三、技术栈

### 3.1 前端技术栈

|技术|用途|
|---|---|
|HTML5|页面结构定义|
|CSS3 + Bootstrap|样式设计与响应式布局|
|JavaScript|前端交互逻辑实现|
|ECharts|数据可视化（图表、地图）|
|jQuery|简化 DOM 操作与 AJAX 请求|
### 3.2 后端技术栈

|技术|用途|
|---|---|
|Python 3.x|核心开发语言|
|Flask|Web 框架，处理 HTTP 请求与路由|
|Flask-CORS|解决跨域请求问题|
|Pandas|数据清洗、统计分析|
|NumPy|数值计算支持|
### 3.3 数据存储

|技术|用途|
|---|---|
|CSV 文件|存储清洗后的岗位数据、分析结果|
|JSON 文件|存储模型配置、匹配策略权重等配置信息|
|Joblib 文件|存储预训练的薪资预测模型|
## 四、数据流程

1. **数据采集**：通过爬虫从 Boss 直聘获取岗位原始数据（HTML 格式）；

2. **数据清洗**：对原始数据进行去重、纠错、格式统一（如薪资字符串转数值、地区名称标准化）；

3. **数据存储**：将清洗后的数据保存为 CSV 文件目录；

4. **数据分析**：后端通过 Pandas 读取 CSV 文件，执行统计分析；

5. **模型应用**：加载 Joblib 格式的预训练模型，处理薪资预测请求；

6. **可视化展示**：前端通过 ECharts 渲染分析结果与图表。
